# End-of-Task Protocol (README Updates)

Before finalizing a task and pushing commits to the remote repository, you must guarantee that the `README.md` is fully up-to-date and serves as the single source of truth for the project.

1. **Pre-Push Requirement**: Ensure the `README.md` file reflects all recent architectural, feature, or structural changes *before* making the final commit to be pushed.
2. **Clear Instructions**:
   - Provide highly detailed, step-by-step setup and execution instructions suitable for first-time users.
   - Detail any prerequisites, necessary environmental configurations, or commands required to run the project.
3. **Disclaimers and Warnings**: Prominently display any disclaimers, warnings, or operational caveats (e.g., local state persistence, data volume warnings, or security notes).
4. **Universal Comprehension**: Format the `README.md` to be a robust, standalone document that is easily understood by users of all technical levels (from beginner to advanced).
